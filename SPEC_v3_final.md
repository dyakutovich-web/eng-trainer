# Часть 1 — Бизнес-аналитик

## 1.1. Продукт и цель
Личный PWA-тренажёр лексики для одного пользователя. Цель v3: **превратить работающий механизм v2
в инструмент, который реально доводит слова до актива и не теряет данные**. Три столпа:
продуктивные упражнения, принудительные интервалы, сохранность прогресса.

## 1.2. Метрики успеха (личные KPI пользователя)
| Метрика | Как считаем | Цель |
|---|---|---|
| Регулярность | стрик дней с ≥1 завершённым блоком | ≥5 дней/нед |
| Пропускная способность | слов, перешедших Этап 1 → Этап 2, в неделю | ≥15 |
| Удержание знаний | % верных в «Тесте выученных» | ≥85% |
| Сохранность | потерянных прогрессов | 0 (есть свежий экспорт) |
Данные для метрик уже есть в Progress/Session-логе; отображение — экран «Списки и прогресс».

## 1.4. Вне скоупа v3
Картинки к словам; синхронизация между устройствами (заменена экспортом/импортом); полная генерация
15k (отдельный процесс, зависит от API-ключа); произношение фраз.

## 1.5. Зависимости и риски
- **US-10/US-12 требуют контента.** Паттерны предлогов и чанки — авторский батч (не ждёт ключа);
  3 примера на sense для полной базы — зависит от генерационного пайплайна (ключ API).
- **Риск потери данных iOS (7-дневное выселение storage)** — закрывается US-05 в первом релизе; до
  него пользователь предупреждён в README.
- **Session-лог для метрик/US-16** — сейчас не пишется; добавить запись ответов (item, тип, верно, ts)
  в Progress-хранилище (лёгкий массив с ротацией 30 дней).

## 1.6. Definition of Done (для каждой US)
Код + юнит-тест на логику (где есть логика) + прогон в браузере + обновление README (если меняется
модель) + запись в CHANGELOG-секцию README.

# Часть 2 — UX/UI

## 2.0. Общие принципы
- **Mobile-first**: все экраны проектируются под 390×844; тач-мишени ≥44px; один столбец.
- **Одна рука**: главные действия — в нижней трети экрана (footer-кнопки уже так работают — сохранить).
- **Никакого стыда**: ошибка = информация. Красный — только подсветка неверного варианта, не полноэкранный.
- **Празднуем блок, не ответ**: микро-фидбэк на ответ лаконичный; сводка блока — главный «момент радости».
- Доступность: aria-live="polite" на фидбэке, aria-label на 🔊/🎤, видимый фокус, contrast ≥4.5:1.

## UX-1. Меню (главный экран) — реструктуризация
Сейчас: 4 кнопки списком + чипы POS. Становится:
```
[🔥 стрик: 4 дня]  [сегодня: 1/1 блок ✓]     ← строка статуса (US-13, в 3.0 — заглушка без стрика)
─────────────────────────────
▶ УЧИТЬ                                        ← главная CTA, самая крупная
   подпись: «активных 27 · доступно сегодня 19»
─────────────────────────────
➕ Пополнить (каталог 75)   🧩 Предлоги (3.1)
✅ Тест выученных (12)      📊 Прогресс
─────────────────────────────
[чипы: Все · Сущ · Глаг · Прил · Нареч · Выражения]
```
- «Доступно сегодня» = слова, у которых кап не исчерпан — управление ожиданиями до входа в сессию.
- Чипы фильтра — внизу, применяются к «Учить» и «Пополнить»; выбранный чип персистится между сессиями.

## UX-2. Упражнение Recall RU→EN (US-01)
- Крупно русский gloss, под ним chip части речи. Поле ввода сразу в фокусе, клавиатура открыта.
- Кнопка «Проверить» + Enter. Ниже — ghost-кнопка «Не помню» (= ошибка −2, показывает ответ; честная
  капитуляция лучше случайного ввода).
- Опечатка ≤1 символа: поле подсвечивается жёлтым, тост «Почти! Проверь написание», ввод остаётся,
  вторая попытка. Второй промах → обычная ошибка.
- После верного ответа: слово + IPA + 🔊 автоозвучка + один пример. Пауза на прочтение — переход по
  кнопке «Дальше», не по таймеру.

## UX-3. Дневной кап — как показываем (US-02)
- В прогресс-барах слова: тип, закрытый на сегодня, получает бейдж «✓ на сегодня» и приглушается.
- Empty-state экран (все пары исчерпаны):
  заголовок «На сегодня всё 🎉», подзаголовок «Ты прошёл всё, что даёт баллы сегодня. Интервалы — это
  и есть запоминание», кнопки: [➕ Добавить новые слова] [Потренировать без баллов] [В меню].
  «Без баллов» — свободная тренировка, очки не пишутся (для желающих продолжить — не запрещаем).

## UX-4. Сводка блока + error review (US-03/04)
Флоу: раунд 20 → экран-прослойка «Разбор ошибок: N слов» (если были) → раунды ревью → сводка.
Сводка блока:
```
Блок пройден!            ← крупно, единственная «праздничная» анимация
✓ 16 верно   ✗ 4 ошибки  ← без процентов-приговоров
Трудные: even · put · depend on      ← тапабельно (карточка слова)
[Ещё блок]  [В меню]
```
- В раундах ревью бейдж «Повтор» в шапке карточки; повторная ошибка — фидбэк без изменения баллов
  («Запомним, вернёмся к нему завтра»).

## UX-5. Трек «Предлоги» (US-12)
- Отдельная плитка меню. Внутри — те же блоки по 20.
- Карточка паттерна (`prep_pattern`), режим ознакомления:
  заголовок-правило («ON — дни и даты»), 3 примера с подсветкой предлога, мини-таблица in/on/at.
- Упражнение gap-fill: предложение с пропуском, 4 кнопки-предлога **в один ряд** (короткие слова —
  ряд из 4 помещается), это быстрый темп — основное удовольствие трека.
- Ввод текстом (Этап 2): поле на 2–4 символа, автофокус.
- Чанки verb+prep в пополнении — бейдж «prep» рядом с леммой.

## UX-6. Экспорт/импорт (US-05)
- В «Прогрессе»: карточка «Резервная копия»: [⬇ Скачать прогресс] [⬆ Восстановить] + строка
  «последний экспорт: 3 дня назад».
- Экспорт = скачивание `vocab-progress-YYYY-MM-DD.json`. Импорт = file-picker + подтверждение
  «заменит текущий прогресс (учу 27, выучено 12) на данные из файла (учу 31, выучено 9)» — с цифрами
  обеих сторон, чтобы не затереть вслепую.
- Мягкий баннер на главном, если экспорту >7 дней: «Давно не было копии» [Скачать] [Позже].

## UX-7. Обновление приложения (US-06)
Баннер поверх меню при готовой новой версии: «Доступно обновление» [Обновить] — по тапу reload с новым
SW. Никаких молчаливых подмен посреди сессии.

## UX-8. Состояния и мелочи
- **Загрузка**: скелетон карточки вместо пустого экрана (данные локальные — редко видно, но без мигания).
- **Ошибка TTS/нет голосов**: у 🔊 состояние disabled с тултипом; упражнение аудирования предлагает
  «Пропустить» сразу.
- **Микрофон запрещён**: произношение автоматически в режиме самооценки с плашкой-объяснением.
- Все деструктивные действия (сброс, импорт) — confirm с цифрами, кнопка отмены первой.
- Тосты — 2.5 с, один одновременно, внизу над footer.
