# TASK — English Vocab Trainer

> Что строим и как поймём, что готово. Полная спецификация — в `README.md`.

## Цель
Собрать рабочую PWA-тренажёр английской лексики для личного использования: SRS + 5 модальностей +
прогрессивная разблокировка, офлайн, с проверкой произношения. База — 15 000 частотных лемм + ~700 выражений.

## Контекст (scoping)
- **Для кого:** Денис (один пользователь, без авторизации).
- **Зачем сейчас:** перевести частотную лексику из пассива в актив (слух/контекст/письмо/речь).
- **Что НЕ входит (v1):** мультипользовательность, синхронизация между устройствами,
  аналитический дашборд (модуль после v1).

## Этапы

### Этап 0 — Снять технический риск (ДО разработки)
- [ ] Проверить **Web Speech API**: TTS и ASR на целевом телефоне/браузере Дениса.
- [ ] Зафиксировать: Web Speech достаточно / нужен fallback Whisper на сервере.
- [ ] **Критерий выхода:** известно, чем делаем произношение и аудио. Без этого UI не начинаем.

### Этап 1 — Контент-пайплайн (`scripts/` → `data/`)
- [ ] Собрать частотный список лемм (COCA/SUBTLEX/wordfreq), POS-тегинг (spaCy), отфильтровать
      топ-15 000 content words, проставить freq_rank и part of speech.
- [ ] Собрать трек выражений: ~150 phrasal verbs (PHaVE) + ~400 коллокаций + ~150 идиом.
- [ ] Пайплайн генерации контента на Item (переводы, IPA, аудио TTS, 2–3 примера, usage notes),
      **траншами по частоте** (первый транш — топ ~1–2k слов + все выражения).
- [ ] Картинки только для конкретных noun/verb.
- [ ] Ревью качества первого транша, сложить в `data/`.
- [ ] **Критерий:** валидный датасет на первый транш + воспроизводимый скрипт догенерации следующих.

### Этап 2 — Ядро SRS (логика, без UI)
- [ ] Модель данных (Item / Progress / Settings / Session) на IndexedDB.
- [ ] Алгоритм веса и выбора карточки (включая freq_factor и guard от повтора).
- [ ] Машина состояний: триаж → ознакомление → закрепление; new→learning→mastered→retired.
- [ ] Прогрессивная разблокировка модальностей (ступени 1→2→3, порог 3).
- [ ] Логика стриков и порогов (10 / 3; ошибка = −3).
- [ ] **Критерий:** юнит-тесты на планировщик и переходы зелёные.

### Этап 3 — Упражнения (UI 5 модальностей)
- [ ] Аудирование · Перевод EN→RU и RU→EN · Контекст · Написание (плитки→с нуля) · Произношение (ASR).
- [ ] Кнопки: «знаю всё», «убрать из ротации», тумблер «эксперт».
- [ ] **Критерий:** все 5 режимов + микс проходятся на реальном датасете.

### Этап 4 — PWA-обвязка и офлайн
- [ ] Service Worker, манифест, установка на телефон, офлайн-кеш контента.
- [ ] Экран настроек (режимы, коэффициенты весов, пороги).
- [ ] **Критерий:** ставится на телефон, работает офлайн, цикл обучения замкнут.

### Этап 5 (после v1) — Аналитика прогресса
- [ ] Дашборд: кривая освоения, слова-застрельщики, ступень застревания, скорость, объём актива,
      продвижение по частотным диапазонам.

## Критерии готовности v1
- На телефоне ставится как PWA, работает офлайн.
- Полный цикл: триаж → ознакомление → закрепление, по 5 модальностям с разблокировкой.
- Слово доходит до `mastered` по правилам; «убрать из ротации» и «знаю всё» работают.
- Произношение проверяется (Web Speech или fallback).
- В ротации частотные слова идут раньше редких.

## Данные и источники
- Частотные леммы: COCA / SUBTLEX / wordfreq + spaCy (POS).
- Выражения: PHaVE (phrasal verbs) + частотные списки коллокаций/идиом.
- AI-генерация контента + TTS (движок выбираем на Этапе 1).
- Под рукой у пользователя: `phrasal verbs.docx`, `Question Words.pdf` (в `~/Desktop/English`) — можно использовать как сверочный материал.
